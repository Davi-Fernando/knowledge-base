# Banco de Dados Relacionais
  É um tipo de banco de dados que organiza as informações em tabelas (relações), compostas por linhas (registros) e colunas (atributos), permitindo o relacionamento entre os dados por meio de campos em comum. Baseia-se na teoria dos conjuntos e na álgebra relacional, possibilitando a manipulação eficiente e estruturada das informações.
<br>

---

## 📌 Conceitos fundamentais<br>

### Tabelas (relações)
Estruturas que armazenam os dados organizados em linhas e colunas.<br>
### Registros (tuplas)
Cada linha da tabela, representando uma instância de dados.<br>
### Atributos (campos)
Colunas da tabela, que descrevem características dos dados.<br>

### Chaves
- Primária (PK): identifica unicamente um registro
- Estrangeira (FK): cria relacionamento entre tabelas<br>

### Consultas (queries)
  Operações realizadas nos dados (ex: SELECT, JOIN, WHERE), cujo resultado é sempre uma tabela virtual.<br>

---

## ⚠️ Pontos importantes
- Os dados são organizados em tabelas relacionadas entre si
- Baseado em teoria dos conjuntos e álgebra relacional
- Utiliza SQL como linguagem principal
- As consultas retornam novos conjuntos de dados (tabelas temporárias)
- Possui independência lógica e física dos dados<br>

## ❌ Erros comuns
- Achar que é apenas uma planilha simples
- Ignorar o uso de chaves primárias e estrangeiras
- Não entender o funcionamento de relacionamentos (JOIN)
- Pensar que os dados são acessados de forma sequencial
- Não modelar corretamente antes da implementação<br>

## 🚀 Aplicação prática
1. Sistemas acadêmicos (controle de alunos e matrículas)
2. Sistemas financeiros (contas, transações, clientes)
3. E-commerce (produtos, pedidos e usuários)<br>


## 💡 Insight
  O grande diferencial do modelo relacional não é só armazenar dados, mas permitir relacioná-los de forma flexível e consultar informações complexas com facilidade, sem depender da estrutura física de armazenamento.
