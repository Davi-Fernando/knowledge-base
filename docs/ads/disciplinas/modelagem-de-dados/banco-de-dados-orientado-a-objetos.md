# Banco de Dados Orientados a Objetos  
  É um tipo de banco de dados que armazena informações na forma de **objetos**, semelhantes aos utilizados em linguagens de programação orientadas a objetos (como Java, C# e C++), combinando **dados (atributos)** e **comportamentos (métodos)** dentro de uma mesma estrutura.

<br>

---

## 📌 Conceitos fundamentais  

### Objetos  
Unidades que representam entidades do sistema, contendo dados e operações.

### Classes  
Modelos que definem a estrutura e comportamento dos objetos.

### Encapsulamento  
Agrupamento de dados e métodos dentro de um objeto.

### Herança  
Permite que uma classe herde características de outra.

### Persistência de objetos  
Capacidade de armazenar objetos de forma permanente no banco.

---

## ⚠️ Pontos importantes
- Armazena dados como **objetos completos**, não apenas tabelas  
- Suporta **tipos de dados complexos** (imagens, mapas, projetos CAD etc.)  
- Integra bem com linguagens orientadas a objetos  
- Surgiu para superar limitações dos bancos relacionais tradicionais  
- Ainda exige **modelagem de dados bem definida**  


## ❌ Erros comuns
- Achar que elimina totalmente a necessidade de modelagem de dados  
- Confundir objeto de programação com estrutura simples de banco  
- Ignorar a complexidade de implementação  
- Supor que substitui totalmente bancos relacionais (na prática, muitas vezes são combinados)

## 🚀 Aplicação prática
1. Sistemas de geoprocessamento (GIS)  
2. Sistemas de engenharia (CAD/CAM/CAE)  
3. Aplicações que trabalham com dados complexos (multimídia, simulações)  

---

## 💡 Insight
  Bancos orientados a objetos aproximam o banco da lógica da aplicação, reduzindo o “gap” entre código e dados — mas, na prática, o mundo real ainda combina fortemente **orientação a objetos + bancos relacionais**, exigindo integração entre os dois modelos.
